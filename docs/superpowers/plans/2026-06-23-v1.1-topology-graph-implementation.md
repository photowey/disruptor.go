# V1.1 Topology Graph Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add named consumer dependency graphs to Disruptor.go without changing the V1 producer publish path.

**Architecture:** `Graph[T]` is a concrete pre-start topology builder in `pkg/disruptor`. `HandleGraph(*Graph[T])` validates and freezes the graph, creates one `BatchEventProcessor` per node, wires barriers from upstream sequences, and enables producer gating only for leaf processors.

**Tech Stack:** Go generics, existing `RingBuffer`, `BatchEventProcessor`, `Barrier`, `Sequence`, standard `testing`, existing examples and benchmark layout.
